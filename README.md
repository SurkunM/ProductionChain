# Управление процессом на производстве по сборке оборудования
Система для автоматизации и контроля сборочного производства. Обеспечивает эффективное распределение задач, отслеживание статусов заказов в реальном времени и управление складскими запасами.

### Ключевые особенности

*   **Real-time оповещения** с использованием SignalR
*   **Динамическое управление задачами** с учетом занятости сотрудников
*   **Интегрированный складской учет** (компоненты и готовая продукция)
*   **Гибкая ролевая модель** (Admin, Manager, Employee)
*   **Полная история производства**

### Технологический стек

### Backend
*   **БД:** MS SQL Server
*   **Data Access:** Repository, Unit of Work, EF Core
*   **Аутентификация:** ASP.NET Core Identity 2.3 (JWT)
*   **Real-time:** SignalR

### Frontend
*   **Фреймворк:** Vue.js
*   **UI-компоненты:** Vuetify

### Роли в системе

| Роль | Описание |
| :--- | :--- |
| **Admin** | Полный доступ к системе |
| **Manager** | Управление заказами, задачами, сотрудниками |
| **Employee** | Выполнение производственных задач |

### Рабочий процесс:

 - Есть закакз (Order), его можно запустить в производство (ProductionOrder). 
    Заказ (Order) может иметь статусы: в работе, в очереди, завершен.
   
 - Руководитель (Role = Manager) может упралять процессом и выдавать задачи сотрудникам (Role = Employee).
    Сотрудники имеют должность и могут быть: занят, свободен, отсутствует.
   
 - При выдаче задачи (AssemblyProductionTask) берутся компоненты для сборки из склада (ComponentsWarehouse).
    Если компонентов хватает, то задача будет выдана.

 - Если сотрудник завершает задачу, то она считаеться выполненой и добавляеться в склад готовой продукции (AssemblyProductionWarehouse).

 - Сотрудник может оповестить менеджера (SignalR) и добавиться в очередь на получение задач (TaskQueue).
    При выдаче задачи или удалении его из очереди задач, сотруднику ему так же выдаеться оповещение с информацией, что задача выдана или что удален из очереди.
   
 - Выполненые задачи сохраняються в таблицу историй (ProductionHistory).
